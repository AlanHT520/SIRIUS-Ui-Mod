📦 SIRIUS Ui Wiki

For SIRIUS Ui v-0.3.2

SIRIUS Ui is a mod that allows you to customize Minecraft's title screen, game selection screen, pause screen, death screen, and settings interface through simple JSON configuration and texture resources. Without writing any Java code, you can easily add images, buttons, backgrounds, sliders, cycle options, lists, catalogs, and other advanced controls.

📌 Currently supported: Title Screen (title_screen), Game Selection Screen (game_selection_screen), Pause Screen (pause_screen), Death Screen (death_screen), and Settings Screen (game_settings_screen). The mod name is SIRIUS Ui.

Overview

SIRIUS Ui replaces hardcoded interfaces with a JSON-based dynamic rendering system. Resource packs can override the following files:

  • assets/sirius_ui/screens/*.json — Screen layouts
  • assets/sirius_ui/buttons/*.json — Button style definitions
  • assets/sirius_ui/slider_styles/*.json — Slider style definitions
  • assets/sirius_ui/cycle_styles/*.json — Cycle button style definitions
  • assets/sirius_ui/catalogs/*.json — Catalog definition files
  • assets/sirius_ui/panels/*.json — Panel/list content definitions
  • Any PNG texture files

Screen Configuration

The root object of each screen JSON is screen, which can contain the following fields:

  • panorama — Background configuration (object or string)
  • graphicsDraw — Array of background layers (optional)
  • elements — Array of interface elements (required)
  • copyright — Copyright text object (only for title screen, optional)

Background (panorama)

Supports three forms:

  • "panorama": "vanilla" — Vanilla dynamic panorama background
  • "panorama": "none" — Solid black background
  • "panorama": { "type": "image", "texture": "namespace:path/image_name" } — Custom image, automatically scaled and centered to fill the screen (preserving aspect ratio).

Background Layers (graphicsDraw)

graphicsDraw is an array where each element defines a colored rectangle drawn between the background and regular elements. Can be used for masks, decorative lines, semi-transparent layers, etc. Each rectangle object contains:

FieldTypeDescription
xstring (expression)X coordinate of the top-left corner
ystring (expression)Y coordinate of the top-left corner
widthstring (expression)Rectangle width
heightstring (expression)Rectangle height
colorstringColor, supports "#RRGGBB", "0xRRGGBB", "0xAARRGGBB" (with alpha)
commentstringOptional comment, not parsed

Elements

The elements array can contain multiple types of objects, each requiring a "type" field.

🖼️ Logo Element (type: "logo")

Used to display static images (e.g., mod logo, decorative image).

FieldTypeRequiredDescription
typestringYesMust be "logo"
texturestringYesTexture identifier, e.g., "sirius_ui:textures/gui/logo.png"
positionobjectYesContains x and y fields (expression strings), defines the top-left corner of the image
sizeobjectYesContains width and height fields (integers), defines the display size in pixels

🔘 Button Element (type: "button")

Interactive button with support for different state textures, text, and actions.

FieldTypeRequiredDescription
typestringYesMust be "button"
stylestringYesButton style file identifier, e.g., "sirius_ui:buttons/default.json"
positionobjectYesx and y expressions, button top-left corner
sizeobjectYeswidth and height (integers), button dimensions
actionobjectYesDefines the behavior when clicked, see Actions section
text_keystringNoOverrides the text key defined in the style file for dynamic button text
enabledbooleanNoDefault true, set to false to disable the button (shows disabled texture and not clickable)

📝 Text Element (type: "text")

Displays text with support for static localization keys, dynamic game data, scaling, shadow, and custom colors.

FieldTypeRequiredDescription
typestringYesMust be "text"
text_keystringConditionalLocalization key name (e.g., "menu.paused"). Required if dynamicType is not provided.
dynamicTypestringConditionalDynamic data type, options:
• "score" — Player's current score (number only)
• "death_cause" — Death cause text (death screen only)
• "player_name" — Player name
If provided without text_key, only the dynamic value is shown; if both are provided, text_key is ignored.
positionobjectYesx and y expressions, text top-left corner (note: text dimensions are determined by the rendered content, elementWidth/elementHeight can be used in expressions)
colorstringNoColor, same format as background layers, default "#FFFFFF" (white)
shadowbooleanNoWhether to draw text shadow, default true
scalefloatNoText scale multiplier, default 1.0. e.g., "scale": 1.5 enlarges by 50%.
💡 Tip: To display vanilla-style "Score: 123", it is recommended to use two text elements: one with "text_key": "deathScreen.score" (outputs "Score: "), and another with "dynamicType": "score" (outputs the number), and position them side by side.

🎚️ Slider Element (type: "slider")

A slider control for adjusting numeric options (e.g., FOV, volume, mouse sensitivity).

FieldTypeDescription
typestringMust be "slider"
option_keystringBound game option key, e.g., "fov", "mouseSensitivity", "soundVolume", "musicVolume", etc.
positionobjectx and y expressions
sizeobjectwidth and height (integers)
stylestringSlider style identifier, e.g., "sirius_ui:slider_styles/default"
enabledbooleanWhether enabled, default true

Example:

{
  "type": "slider",
  "option_key": "fov",
  "position": { "x": "width/2 - 100", "y": "height/2" },
  "size": { "width": 200, "height": 20 },
  "style": "sirius_ui:slider_styles/default"
}

🔄 Cycle Element (type: "cycle")

A cycle button for selecting enumerated options (e.g., graphics quality, cloud rendering, particle effects).

FieldTypeDescription
typestringMust be "cycle"
option_keystringBound game option key, e.g., "graphicsMode", "cloudStatus", "particles", "chatVisibility", "narrator", "mainHand", etc.
positionobjectx and y expressions
sizeobjectwidth and height (integers)
stylestringCycle style identifier, e.g., "sirius_ui:cycle_styles/default"
enabledbooleanWhether enabled, default true

Example:

{
  "type": "cycle",
  "option_key": "graphicsMode",
  "position": { "x": "width/2 - 100", "y": "height/2 + 30" },
  "size": { "width": 200, "height": 20 },
  "style": "sirius_ui:cycle_styles/default"
}

📋 List Element (type: "list")

A scrollable list container that can hold any type of child elements (buttons, text, sliders, etc.). Supports automatic vertical layout.

FieldTypeDescription
typestringMust be "list"
idstringOptional, used for switch_panel action targeting
positionobjectx and y expressions
sizeobjectwidth and height (integers), defining the visible area dimensions
sourcestringIdentifier of the list content definition file, e.g., "sirius_ui:panels/video_settings"
stylestringOptional, list background style (button style identifier)
enabledbooleanWhether enabled, default true

Structure of a list content file (e.g., panels/video_settings.json):

{
  "layout": "vertical",
  "item_spacing": 6,
  "padding": { "top": 10, "left": 10, "right": 10, "bottom": 10 },
  "items": [ ... ]
}
  • layout: Currently only supports "vertical", child elements are automatically arranged downwards.
  • item_spacing: Vertical spacing between child elements (in pixels).
  • padding: Inner padding, each direction can be omitted.
  • items: Array of child elements, same types as screen elements (but position.y is automatically calculated by the layout).

Example:

{
  "type": "list",
  "id": "content_panel",
  "position": { "x": "width/2 + 20", "y": "height/4" },
  "size": { "width": 260, "height": 300 },
  "source": "sirius_ui:panels/video_settings"
}

🗂️ Catalog Element (type: "catalog")

A directory/menu bar, typically used for left-side navigation in settings interfaces. Contains multiple catalog items; clicking one switches the content of the right panel.

FieldTypeDescription
typestringMust be "catalog"
idstringOptional, catalog identifier
positionobjectx and y expressions
sizeobjectwidth and height (integers)
sourcestringIdentifier of the catalog definition file, e.g., "sirius_ui:catalogs/settings_catalog"
stylestringOptional, catalog background style
enabledbooleanWhether enabled, default true

Structure of a catalog definition file (e.g., catalogs/settings_catalog.json):

{
  "layout": "vertical",
  "item_spacing": 2,
  "padding": { "top": 5, "left": 5, "right": 5, "bottom": 5 },
  "items": [
    {
      "text_key": "options.video",
      "action": { "type": "switch_panel", "target": "content_panel", "content": "sirius_ui:panels/video_settings" }
    },
    ...
  ],
  "selected_style": "sirius_ui:catalog_item_selected"
}
  • items: Array of catalog items, each containing text_key, action (usually switch_panel), optional icon and style.
  • selected_style: Highlight style for the selected item (optional).

Example:

{
  "type": "catalog",
  "id": "settings_catalog",
  "position": { "x": "width/4 - 120", "y": "height/4" },
  "size": { "width": 200, "height": 300 },
  "source": "sirius_ui:catalogs/settings_catalog"
}

Copyright Info (copyright)

Only for title screen, used to display copyright text at the bottom of the screen. Object fields:

FieldTypeDescription
textKeystringLocalization key name
positionobjectx and y expressions, defines the text top-left corner
colorstringColor, default "#808080"

Button Style

Button style files (e.g., buttons/default.json) must contain a top-level button object with the following fields:

FieldTypeDescription
text_keystringDefault localization key for button text (can be overridden by text_key in button element)
textureobjectContains normal, hovered, disabled fields, values are texture identifiers (without .png), textures should be placed in assets/namespace/textures/gui/sprites/.
text_colorstring or intNormal state text color, recommended to use string like "#FFFFFF"
hover_text_colorstring or intHovered state text color
disabled_text_colorstring or intDisabled state text color
text_offset_xstring (expression)X offset of text relative to button top-left, default "center" (horizontal center)
text_offset_ystring (expression)Y offset of text relative to button top-left, recommended to use "(height - elementHeight)/2" for vertical centering
📌 Note: Button textures use nine-slice scaling. The texture file needs gui metadata (e.g., gui_metadata.json) or uses default stretching. Vanilla button textures are already configured for nine-slice.

Located in the cycle_styles/ directory, structure:

Actions

The action field of a button is an object containing a type and specific parameters. The following action types are supported:

typeParametersDescription
open_screenscreenIdOpens the specified screen. Predefined screenIds include: singleplayer, multiplayer, options, realms, selectGame, language, accessibility, advancements, stats, share_to_lan, player_social, etc.
quit_gameNoneQuits the game
close_screenNoneCloses the current screen and returns to the parent screen (if any)
resume_gameNoneResumes the game, closes the pause screen and grabs the mouse (for pause screen)
disconnectNoneDisconnects from the current server and returns to the title screen
open_linkurlOpens an external link, shows a confirmation dialog first. Example: { "type": "open_link", "url": "https://example.com" }
respawnNoneRespawns the player (for death screen)
exit_to_titleNoneExits to the title screen (for death screen)
switch_paneltarget (id of target List), content (new content definition file identifier)Switches the content panel of the specified list. Usually triggered by a catalog item.

Coordinate System & Expressions

Origin

The origin (0,0) is at the top-left corner of the screen. X increases to the right, Y increases downwards.

▲ Origin is at the top-left corner, X goes right, Y goes down.

Element Anchor

The position field defines the top-left corner of the element, not its center.

⚠️ Common misconception: "x": "width/2", "y": "height/2" does not center the element — it only places the top-left corner at the screen center.

Correct centering expression:

"x": "(width - elementWidth)/2"
"y": "(height - elementHeight)/2"

Expression Variables & Operations

Expressions support the following variables:

  • width — Current screen width (scaled pixels)
  • height — Current screen height
  • elementWidth — Element's own width (for buttons, from size.width; for text, the rendered text width, affected by scale)
  • elementHeight — Element's own height (similarly)

Inside list/catalog child elements, the following are also available:

  • containerWidth — Width of the container (List/Catalog)
  • containerHeight — Height of the container

Supported operators: + - * / ( ), division is integer truncation. Shortcuts:

  • "center" → "(width - elementWidth)/2" (for x coordinate only)
  • "bottom-20" → "height - elementHeight - 20" (for y coordinate only)

Configuration Example

An example of a settings screen containing a catalog and a list:

{
  "screen": {
    "panorama": { "type": "vanilla" },
    "elements": [
      {
        "type": "catalog",
        "id": "settings_catalog",
        "position": { "x": "width/4 - 120", "y": "height/4" },
        "size": { "width": 200, "height": 300 },
        "source": "sirius_ui:catalogs/settings_catalog"
      },
      {
        "type": "list",
        "id": "content_panel",
        "position": { "x": "width/2 + 20", "y": "height/4" },
        "size": { "width": 260, "height": 300 },
        "source": "sirius_ui:panels/general_settings"
      },
      {
        "type": "button",
        "text_key": "gui.done",
        "position": { "x": "width/2 - 100", "y": "height - 40" },
        "size": { "width": 200, "height": 20 },
        "style": "sirius_ui:buttons/default",
        "action": { "type": "close_screen" }
      }
    ]
  }
}

Troubleshooting

ProblemPossible Cause & Solution
Button not respondingCheck if action and screenId are correct; ensure enabled is true (default is true).
Texture shows pink/black checkerboardPath error or file missing; verify the texture file is in the correct namespace and path, and the .png extension is not included in the identifier.
Expression error "Unknown variable"Only width, height, elementWidth, elementHeight are allowed; check spelling.
Background layer not showingCheck color format (e.g., "#RRGGBB" or "0xAARRGGBB") and that coordinates/size are within screen bounds.
Custom panorama not showingConfirm panorama.type is "image", texture path includes correct namespace and file, and the image file exists.
Text color not showing or transparentEnsure color value is a string, e.g., "#FFFFFF"; if using 0xFFFFFF format, it must be quoted and include 0x.
Dynamic text (score/death cause) not showingConfirm dynamicType spelling is correct and the current screen supports that type (e.g., death_cause only works on death screen).
Interface not updating after JSON changeSince version 0.3.0, reopening the screen applies changes without restarting. If still not working, press F3+T to reload resource packs.
Button text not centeredCheck if text_offset_y in the style file is set to "(height - elementHeight)/2".
Slider does not display valueCheck if show_value_text in the style file is set to true.
List content does not scrollEnsure the total content height exceeds the list height, and the mouse is scrolling within the list area.
Panel switching does not workVerify that the target of switch_panel matches the id of a List element, and content points to a valid panel definition file.
🔧 Press F3+T to quickly reload the resource pack — very convenient for debugging layouts.
