📦 SIRIUS Ui Wiki

For Minecraft 26.1.1 | Data‑Driven UI Framework (In Development)

SIRIUS Ui is a mod that lets you customise Minecraft's Title Screen and Game Selection Screen using simple JSON configurations and texture resources. No Java coding required – add images, buttons, backgrounds, and dynamic layouts with ease.

📌 Currently supports two screens: Title Screen (title_screen) and Game Selection Screen (game_selection_screen). The mod is named SIRIUS Ui.

Overview

SIRIUS Ui replaces the hard‑coded Title and Game Selection screens with a JSON‑driven dynamic rendering system. A resource pack can override the following files:

  • assets/sirius_ui/screens/title_screen.json — Title Screen layout
  • assets/sirius_ui/screens/game_selection_screen.json — Game Selection Screen layout
  • assets/sirius_ui/buttons/*.json — Button style definitions
  • Any PNG texture files

Core features:

  • Image display (logo element)
  • Interactive buttons (normal / hovered / disabled textures)
  • Dynamic expression coordinates (based on screen size and element dimensions)
  • Button actions: open screens, quit game, go back
  • Background panorama control (vanilla, none, custom image)
  • Custom background layers (graphicsDraw) – coloured rectangles with expression support
  • Static text element (text)

Resource Pack Structure

Your resource pack must follow this exact folder hierarchy:

📦 your_resource_pack/
└── 📁 assets/
    └── 📁 sirius_ui/
        ├── 📁 screens/
        │   ├── 📄 title_screen.json
        │   └── 📄 game_selection_screen.json
        ├── 📁 buttons/
        │   ├── singleplayer.json
        │   ├── multiplayer.json
        │   └── ...
        └── 📁 textures/
            └── 📁 gui/
                ├── custom_logo.png
                ├── btn_normal.png
                └── ...
└── 📄 pack.mcmeta

pack.mcmeta example:

{
  "pack": {
    "pack_format": 26,
    "description": "SIRIUS Ui custom interface pack"
  }
}

Screen Configuration

The root object of each screen JSON is screen, which may contain:

  • panorama — background configuration
  • graphicsDraw — array of background layers
  • elements — array of UI elements
  • copyright — copyright text (Title Screen only)

Background (panorama)

Three forms are supported:

  • "panorama": "vanilla" — vanilla dynamic panorama
  • "panorama": "none" — solid black background
  • "panorama": { "type": "image", "texture": "sirius_ui:textures/gui/background.png" } — custom image (scaled to fill, centered)

Elements (elements)

The elements array can contain objects of these types:

  • logo : displays a static image. Required fields: texture, position, size.
  • button : interactive button. Required fields: style, position, size, action. Optional: enabled (default true), text_key (overrides style text).
  • text : static text. Required fields: text_key, position. Optional: color (default 0xFFFFFF), shadow (default true).

Background Layers (graphicsDraw)

graphicsDraw is an array of coloured rectangles drawn between the background and normal elements. Each layer supports expressions for all fields:

  • x, y — top‑left corner
  • width, height — dimensions
  • color — colour string: "#RRGGBB", "0xRRGGBB", or "0xAARRGGBB"

Example: a red decorative line at the bottom

{
  "x": "0",
  "y": "height - 4",
  "width": "width",
  "height": "4",
  "color": "#AA0000"
}

Button Style (button style)

Button style files (e.g., singleplayer.json) must have a top‑level button object:

{
  "button": {
    "text_key": "menu.singleplayer",
    "texture": {
      "normal": "minecraft:widget/button",
      "hovered": "minecraft:widget/button_highlighted",
      "disabled": "minecraft:widget/button_disabled"
    },
    "text_color": 16777215,
    "hover_text_color": 16777088,
    "disabled_text_color": 10526880,
    "text_offset_x": "center",
    "text_offset_y": "(height - elementHeight)/2"
  }
}
  • text_key — localization key shown on the button.
  • texture — identifiers for the three states (no .png extension; textures must be placed in assets/namespace/textures/gui/sprites/).
  • text_color / hover_text_color / disabled_text_color — ARGB colours as decimal integers.
  • text_offset_x / text_offset_y — text offset from the button's top‑left corner (expressions allowed). You can also use hovered_text_offset_* and disabled_text_offset_* for state‑specific offsets.

Actions (action)

The button action field supports the following types:

  • Open screen: { "type": "open_screen", "screenId": "singleplayer" }
    Predefined screenId values:
    • singleplayer — singleplayer world selection
    • multiplayer — multiplayer screen
    • options — options screen
    • realms — Realms main screen
    • selectGame — custom game selection screen (provided by the mod)
    • language — language settings
    • accessibility — accessibility settings
  • Quit game: { "type": "quit_game" }
  • Close current screen (go back): { "type": "close_screen" }

Coordinate System & Expressions

Origin

The origin (0,0) is at the top‑left corner of the screen. X increases to the right, Y increases downward.

▲ Origin at top‑left, X right, Y down.

Element Anchor

The position field defines the top‑left corner of the element, not its center.

⚠️ Common mistake: "x": "width/2", "y": "height/2" does not center the element – it places the top‑left corner at the screen center.

Correct centering expressions:

"x": "(width - elementWidth)/2"
"y": "(height - elementHeight)/2"

Expression Variables & Operations

Available variables:

  • width — screen width (scaled)
  • height — screen height
  • elementWidth — element width (from size.width)
  • elementHeight — element height (from size.height)

Operators: + - * / ( ); division is integer truncation. Shortcuts:

  • "center" → "(width - elementWidth)/2" (only for element x)
  • "bottom-20" → "height - elementHeight - 20" (only for element y)

Aligning Different‑sized Elements

  • Left / right / horizontal center: use the same x expression for all elements.
  • Bottom alignment: use "y": "baseY - elementHeight", where baseY is the desired baseline.
  • Vertical stack: manually accumulate y values, e.g., "y": "100 + 20 + 10" (first button height 20, gap 10).
Core formula: top‑left corner = target reference point - element size offset (e.g., for right alignment: reference point = width, offset = elementWidth).

Configuration Example

Complete Title Screen configuration (dark theme, custom logo, three buttons, top title):

{
  "screen": {
    "panorama": { "type": "image", "texture": "sirius_ui:textures/gui/background.png" },
    "graphicsDraw": [
      { "x": "0", "y": "0", "width": "width", "height": "height", "color": "#0A0A0A" },
      { "x": "0", "y": "height - 4", "width": "width", "height": "4", "color": "#FFAA00" }
    ],
    "elements": [
      {
        "type": "text",
        "text_key": "auui.title",
        "position": { "x": "center", "y": 12 },
        "color": 16777215,
        "shadow": true
      },
      {
        "type": "logo",
        "texture": "sirius_ui:textures/gui/fancy_logo.png",
        "position": { "x": "center", "y": 40 },
        "size": { "width": 300, "height": 90 }
      },
      {
        "type": "button",
        "text_key": "menu.singleplayer",
        "style": "sirius_ui:buttons/custom_play.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2 - 40" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "open_screen", "screenId": "singleplayer" }
      },
      {
        "type": "button",
        "text_key": "menu.multiplayer",
        "style": "sirius_ui:buttons/custom_multi.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "open_screen", "screenId": "multiplayer" }
      },
      {
        "type": "button",
        "text_key": "menu.quit",
        "style": "sirius_ui:buttons/custom_quit.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2 + 40" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "quit_game" }
      }
    ]
  }
}

Troubleshooting

IssuePossible cause & solution
Button does nothingCheck action and screenId; ensure enabled is true.
Texture shows pink/black checkerboardWrong path or missing file; verify namespace and .png extension.
Expression error “Unknown variable”Only width, height, elementWidth, elementHeight are allowed; check spelling.
Background layer not shownCheck colour format (e.g., #RRGGBB) and coordinate range.
Custom panorama image not shownConfirm panorama.type is "image", texture path includes .png, and file exists.
