📦 SIRIUS Ui 维基

适用 Minecraft 26.1.1 | 数据驱动型界面框架（开发中）

SIRIUS Ui 是一款模组，允许您通过简洁的 JSON 配置与纹理资源自定义 Minecraft 的标题画面与游戏选择画面。无需编写 Java 代码，即可轻松添加图像、按钮、背景以及动态布局。

📌 目前支持两种屏幕：标题画面（title_screen）与游戏选择画面（game_selection_screen）。模组名称为 SIRIUS Ui。

总览

SIRIUS Ui 将硬编码的标题画面与游戏选择画面替换为基于 JSON 的动态渲染系统。资源包可覆写下列文件：

  • assets/sirius_ui/screens/title_screen.json — 标题画面布局
  • assets/sirius_ui/screens/game_selection_screen.json — 游戏选择画面布局
  • assets/sirius_ui/buttons/*.json — 按钮样式定义
  • 任意 PNG 纹理文件

核心特性：

  • 图像显示（logo 元素）
  • 可交互按钮（常态 / 悬停 / 禁用纹理）
  • 基于表达式的动态坐标（基于屏幕尺寸与元素尺寸）
  • 按钮动作：打开屏幕、退出游戏、返回
  • 背景全景控制（原版、无、自定义图像）
  • 自定义背景图层（graphicsDraw）—— 支持表达式的彩色矩形
  • 静态文本元素（text）

资源包结构

您的资源包必须遵循下述精确的文件夹层次：

📦 您的资源包/
└── 📁 assets/
    └── 📁 sirius_ui/
        ├── 📁 screens/
        │   ├── 📄 title_screen.json
        │   └── 📄 game_selection_screen.json
        ├── 📁 buttons/
        │   ├── singleplayer.json
        │   ├── multiplayer.json
        │   └── ...
        └── 📁 textures/
            └── 📁 gui/
                ├── custom_logo.png
                ├── btn_normal.png
                └── ...
└── 📄 pack.mcmeta

pack.mcmeta 示例：

{
  "pack": {
    "pack_format": 26,
    "description": "SIRIUS Ui 自定义界面资源包"
  }
}

屏幕配置

每个屏幕 JSON 的根对象为 screen，其可包含：

  • panorama — 背景配置
  • graphicsDraw — 背景图层数组
  • elements — 界面元素数组
  • copyright — 版权文本（仅标题画面）

背景（panorama）

支持三种形式：

  • "panorama": "vanilla" — 原版动态全景
  • "panorama": "none" — 纯黑背景
  • "panorama": { "type": "image", "texture": "sirius_ui:textures/gui/background.png" } — 自定义图像（缩放填充，居中）

元素（elements）

elements 数组可包含以下类型的对象：

  • logo ：显示静态图像。必填字段：texture、position、size。
  • button ：可交互按钮。必填字段：style、position、size、action。可选：enabled（默认为 true）、text_key（覆写样式的文本）。
  • text ：静态文本。必填字段：text_key、position。可选：color（默认为 0xFFFFFF）、shadow（默认为 true）。

背景图层（graphicsDraw）

graphicsDraw 是一组彩色矩形数组，绘制于背景与常规元素之间。每个图层均支持表达式：

  • x、y — 左上角坐标
  • width、height — 宽高尺寸
  • color — 颜色字符串："#RRGGBB"、"0xRRGGBB" 或 "0xAARRGGBB"

示例：底部一条红色装饰线

{
  "x": "0",
  "y": "height - 4",
  "width": "width",
  "height": "4",
  "color": "#AA0000"
}

按钮样式（button style）

按钮样式文件（例如 singleplayer.json）必须包含顶层对象 button：

{
  "button": {
    "text_key": "menu.singleplayer",
    "texture": {
      "normal": "minecraft:widget/button",
      "hovered": "minecraft:widget/button_highlighted",
      "disabled": "minecraft:widget/button_disabled"
    },
    "text_color": 16777215,
    "hover_text_color": 16777088,
    "disabled_text_color": 10526880,
    "text_offset_x": "center",
    "text_offset_y": "(height - elementHeight)/2"
  }
}
  • text_key — 显示在按钮上的本地化键名。
  • texture — 三种状态的纹理标识符（不含 .png 扩展名；纹理须置于 assets/namespace/textures/gui/sprites/ 目录）。
  • text_color / hover_text_color / disabled_text_color — 十进制整数表示的 ARGB 颜色值。
  • text_offset_x / text_offset_y — 文本相对于按钮左上角的偏移量（允许表达式）。您也可以使用 hovered_text_offset_* 和 disabled_text_offset_* 来实现状态相关的偏移。

动作（action）

按钮的 action 字段支持以下类型：

  • 打开屏幕：{ "type": "open_screen", "screenId": "singleplayer" }
    预定义的 screenId 取值：
    • singleplayer — 单人游戏世界选择
    • multiplayer — 多人游戏屏幕
    • options — 选项屏幕
    • realms — Realms 主屏幕
    • selectGame — 模组提供的自定义游戏选择屏幕
    • language — 语言设置
    • accessibility — 辅助功能设置
  • 退出游戏：{ "type": "quit_game" }
  • 关闭当前屏幕（返回）：{ "type": "close_screen" }

坐标系与表达式

原点

原点 (0,0) 位于屏幕左上角。X 轴向右递增，Y 轴向下递增。

▲ 原点位于左上角，X 向右，Y 向下。

元素锚点

position 字段定义的是元素的左上角，而非其中心。

⚠️ 常见误区："x": "width/2", "y": "height/2" 并不能将元素居中——它只是将元素的左上角置于屏幕中心。

正确的居中表达式：

"x": "(width - elementWidth)/2"
"y": "(height - elementHeight)/2"

表达式变量与运算

可用变量：

  • width — 屏幕宽度（缩放后）
  • height — 屏幕高度
  • elementWidth — 元素宽度（来自 size.width）
  • elementHeight — 元素高度（来自 size.height）

运算符：+ - * / ( )；除法为整数截断。快捷写法：

  • "center" → "(width - elementWidth)/2"（仅用于元素 x 坐标）
  • "bottom-20" → "height - elementHeight - 20"（仅用于元素 y 坐标）

常用对齐公式

效果X 表达式Y 表达式
水平居中"center" 或 "(width - elementWidth)/2"任意（保持原值）
垂直居中任意"(height - elementHeight)/2"
完全居中"center" 或 "(width - elementWidth)/2""(height - elementHeight)/2"
左上边距 10px1010
右上（距右边缘 20px）"width - elementWidth - 20"20
左下（距底边缘 30px）20"bottom-30" 或 "height - elementHeight - 30"
右下"width - elementWidth - 15""height - elementHeight - 15"

不同尺寸元素的对齐

  • 左 / 右 / 水平居中：对所有元素使用相同的 x 表达式。
  • 底部对齐：使用 "y": "baseY - elementHeight"，其中 baseY 为目标基线。
  • 垂直堆叠：手动累加 y 值，例如 "y": "100 + 20 + 10"（首个按钮高度 20，间距 10）。
核心公式：左上角坐标 = 目标参考点 - 元素尺寸偏移（例如右对齐：参考点 = width，偏移 = elementWidth）。

配置范例

完整的标题画面配置（深色主题、自定义徽标、三个按钮、顶部标题）：

{
  "screen": {
    "panorama": { "type": "image", "texture": "sirius_ui:textures/gui/background.png" },
    "graphicsDraw": [
      { "x": "0", "y": "0", "width": "width", "height": "height", "color": "#0A0A0A" },
      { "x": "0", "y": "height - 4", "width": "width", "height": "4", "color": "#FFAA00" }
    ],
    "elements": [
      {
        "type": "text",
        "text_key": "auui.title",
        "position": { "x": "center", "y": 12 },
        "color": 16777215,
        "shadow": true
      },
      {
        "type": "logo",
        "texture": "sirius_ui:textures/gui/fancy_logo.png",
        "position": { "x": "center", "y": 40 },
        "size": { "width": 300, "height": 90 }
      },
      {
        "type": "button",
        "text_key": "menu.singleplayer",
        "style": "sirius_ui:buttons/custom_play.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2 - 40" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "open_screen", "screenId": "singleplayer" }
      },
      {
        "type": "button",
        "text_key": "menu.multiplayer",
        "style": "sirius_ui:buttons/custom_multi.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "open_screen", "screenId": "multiplayer" }
      },
      {
        "type": "button",
        "text_key": "menu.quit",
        "style": "sirius_ui:buttons/custom_quit.json",
        "position": { "x": "center", "y": "(height - elementHeight)/2 + 40" },
        "size": { "width": 200, "height": 24 },
        "action": { "type": "quit_game" }
      }
    ]
  }
}

疑难排解

问题可能原因与解决方案
按钮无响应检查 action 与 screenId；确保 enabled 为 true。
纹理显示粉黑棋盘格路径错误或文件缺失；确认命名空间与 .png 扩展名。
表达式错误“未知变量”仅允许 width, height, elementWidth, elementHeight；检查拼写。
背景图层未显示检查颜色格式（例如 #RRGGBB）及坐标范围。
自定义全景图未显示确认 panorama.type 为 "image"，texture 路径包含 .png，且文件存在。
